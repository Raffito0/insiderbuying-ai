# Metodologia: Psychological Framework per qualsiasi business

**Scopo**: Processo replicabile per costruire il framework psicologico completo di qualsiasi prodotto/servizio/business — dalla ricerca alla strategia al copy finale.

---

## FASE 0 — Input (cosa serve per iniziare)

Prima di tutto, rispondi a queste 5 domande:

```
1. COSA VENDI: [prodotto/servizio — 1 frase]
2. A CHI: [target — età, reddito, comportamento, cosa usano ora]
3. QUANTO COSTA: [pricing tiers]
4. COSA ESISTE GIÀ: [sito live? landing page? niente?]
5. OBIETTIVO: [vendere abbonamenti? lead generation? download? signup?]
```

---

## FASE 1 — Deep Research (3 ricerche parallele)

Lancia 3 ricerche contemporaneamente. Ognuna produce un documento separato.

### Ricerca 1: Landscape competitivo

```
Per ogni competitor trovato, documenta:
- Nome + URL
- Cosa offre (free vs paid, feature)
- Pricing esatto
- Come si posiziona (headline, tagline, tono)
- Debolezze (review negative, Reddit complaints, limitazioni)
- Cosa NON ha che tu hai

Output: tabella con GAP ANALYSIS — quali buchi nel mercato il tuo prodotto riempie
```

**Dove cercare**: Google "[categoria] tool/app/service", Reddit "best [categoria]", Product Hunt, G2/Capterra review, Twitter/X complaints.

### Ricerca 2: Psicologia del target

```
Documenta:
- Cosa li tiene svegli la notte (paure, frustrazioni, ansie)
- Perché pagano per prodotti simili (motivazioni reali, non dichiarate)
- Perché cancellano/abbandonano (review negative competitor)
- Quanto sono sensibili al prezzo (range accettabile nel settore)
- Come percepiscono il segnale di "qualità" vs "scam" nel tuo settore
- Linguaggio esatto che usano (frasi da Reddit, forum, review)

Output: 3 segmenti di utente con: cerca/pain/paga perché/obiezione principale
```

**Dove cercare**: Reddit (subreddit di settore), Trustpilot/BBB review dei competitor, forum di settore, studi accademici se esistono, survey pubbliche.

### Ricerca 3: Pattern di copy che funzionano nel settore

```
Documenta:
- Come i leader del settore si posizionano (headline, tono, struttura)
- Quali parole segnalano "autorità" vs "amatoriale" nel tuo settore
- Quali CTA convertono meglio (dati se disponibili, pattern se no)
- Come gestiscono obiezioni (FAQ, disclaimer, trust signal)
- Pricing page patterns (anchoring, naming, framing)
- Trust signals specifici del settore (certificazioni, fonti dati, metodologia)

Output: lista di pattern concreti con esempi reali da siti competitor
```

**Dove cercare**: landing page dei top 5 competitor, case study di settore, articoli su CRO/copywriting specifici per il settore.

## FASE 2 — Sintesi strategica

Dalla ricerca emergono le risposte a queste domande fondamentali:

### 2A — Il Core Insight (1 frase)

```
Qual è il VERO problema che risolvi?
Non il problema tecnico — il problema EMOTIVO.

Formula: "[Il target] non ha bisogno di [cosa sembra].
Ha bisogno di [cosa vuole veramente]."

Esempio earlyinsider:
"I retail investor non hanno bisogno di più dati.
Hanno bisogno di sapere quello che i CEO sanno, quando lo sanno."
```

### 2B — I 3 Pilastri Psicologici

Ogni business ha esattamente 3 leve psicologiche principali. Non di più.

```
Per ogni pilastro:
- NOME: frase che lo riassume (sarà il tuo mantra interno)
- MECCANISMO COGNITIVO: quale bias/principio psicologico sfrutti
- COSA NON È: il confine — cosa NON dire per non cadere nel manipolativo
- COSA È: come si manifesta nel copy
- DOVE APPARE: in quali sezioni/pagine del sito

Come trovarli:
1. Guarda i 3 GAP principali dalla ricerca competitiva
2. Per ogni gap, chiediti: "quale emozione attiva nel target?"
3. Quella emozione → quel pilastro

Pilastri comuni per tipo di business:
- SaaS B2B: Efficienza + Autorità + Risk Reduction
- Consumer app: Status + Appartenenza + Gratificazione
- Fintech: Information Edge + Trust + Urgenza temporale
- Health/wellness: Controllo + Trasformazione + Prova sociale
- Education: Competenza + FOMO + Identità aspirazionale
```

### 2C — Il Tone of Voice

```
Definisci:
- OGNI frase deve contenere: [cosa? numero/fatto/nome/dato]
- Competenza assunta: [cosa NON spieghi perché il target già sa]
- Mai: [lista di parole/frasi bandite — specifiche per il settore]
- Sempre: [lista di parole che segnalano autorità nel settore]
- Framing: [loss framing dove? gain framing dove?]

Formula per trovare il tono:
"Se il nostro brand fosse una persona, sarebbe un [ruolo]
che parla a un [ruolo del target] — come un [relazione]."

Esempio earlyinsider:
"Un senior analyst di Goldman che parla a un investitore esperto —
come un collega più senior, non come un venditore."
```

## FASE 3 — Trust Architecture

Ogni settore ha i suoi trust signals. Identifica i 5 layer:

```
LAYER 1 — Source Authority
"Da dove vengono i tuoi dati/la tua competenza?"
→ Qual è il proxy di credibilità più forte nel tuo settore?
→ Esempio finanza: SEC EDGAR. Esempio salute: studi clinici. Esempio SaaS: loghi clienti.

LAYER 2 — Specificity as Proof
"I numeri sono specifici o arrotondati?"
→ $4.2B > "miliardi". 17,325+ > "migliaia". 47 secondi > "in tempo reale".
→ Numeri dispari e specifici = misurati. Numeri tondi = inventati.

LAYER 3 — External Validation
"Chi altro dice che sei bravo?"
→ Studi accademici, press mentions, review verificate, loghi clienti.
→ Nel tuo settore, quale validazione esterna ha più peso?

LAYER 4 — Stated Limitations
"Cosa NON puoi fare?"
→ Dichiarare limiti AUMENTA trust (pattern paper scientifico).
→ Il visitatore pensa "se dichiarano i limiti, il resto è vero."

LAYER 5 — Disclaimer as Positioning
"Il disclaimer è nascosto o è parte del brand?"
→ "Non siamo financial advisor" POSIZIONA come tool professionale.
→ "Non siamo una dieta miracolosa" POSIZIONA come approccio serio.
→ Il disclaimer diventa un vantaggio, non una liability.
```

## FASE 4 — Conversion Architecture

Mappa il percorso psicologico del visitatore in 4 gate:

```
GATE 1 — COMPREHENSION (0-5 secondi)
Domanda del visitatore: "Cos'è?"
Risponde: Hero (headline + subheadline + trust line)
Test: qualcuno che non conosce il tuo settore capisce cosa fai in 5 secondi?

GATE 2 — PROOF (5-60 secondi)
Domanda: "È vero?"
Risponde: Social proof, numeri, esempio prodotto, citazioni
Test: il visitatore crede che il prodotto funziona?

GATE 3 — VALUE (60-180 secondi)
Domanda: "Vale il prezzo?"
Risponde: Pricing con anchoring, loss framing, confronto alternativa
Test: il visitatore vede il prezzo come ovvio, non come ostacolo?

GATE 4 — ACTION (signup/checkout)
Domanda: "Sto facendo un errore?"
Risponde: Conferma decisione, zero friction, trust signals finali
Test: il form ha il minimo di campi possibile? Le paure sono eliminate?
```

## FASE 5 — Page-by-Page Strategy

Per OGNI pagina del sito, compila questa scheda:

```
PAGINA: [nome]
URL: [/path]

JOB (1 frase): cosa deve succedere quando il visitatore lascia questa pagina?

TRIGGER PRIMARIO: [quale dei 3 pilastri è dominante qui?]
TRIGGER SECONDARIO: [quale pilastro supporta?]
BIAS COGNITIVO: [quale bias stai sfruttando?]

SEZIONI (in ordine):
1. [Nome sezione] — Psicologia: [meccanismo] — Copy direction: [cosa dire]
2. [Nome sezione] — Psicologia: [meccanismo] — Copy direction: [cosa dire]
...

ERRORI DA EVITARE su questa pagina:
- [cosa NON fare]
```

**Pagine standard da coprire**:
- Homepage (comprehension → pricing)
- Pricing (quale piano, non se comprare)
- About (chi siamo senza dire chi siamo)
- How It Works (competenza dimostrata)
- FAQ (obiezioni eliminate)
- Lead Magnet / Free Report (email capture)
- Signup (conferma decisione)
- Blog / Content (autorità + SEO)

## FASE 6 — Naming Strategy

```
TIER NAMING:
- I nomi dei plan descrivono il PRODOTTO o l'UTENTE?
- Feature-based (Basic/Pro/Enterprise): descrivono il prodotto → più deboli
- Identity-based (Monitor/Analyst/Investor): descrivono l'utente → più forti
- L'utente si chiede "quale sono IO?" non "quali feature voglio?"

COME SCEGLIERE:
1. Lista 3 identità che il tuo target vorrebbe avere
2. Mappale ai tier dal più accessibile al più aspirazionale
3. Verifica: un utente del tier 1 VUOLE diventare tier 3? Se sì, i nomi funzionano

REPORT/PRODUCT NAMING:
- Il nome è un FINDING o un FORMATO?
- "Backtest Report" = formato (debole)
- "The CEO Alpha Report" = finding (forte)
- Formula: "[The] [Aggettivo] [Nome del concetto]"
```

## FASE 7 — Emotional Journey Map

```
Mappa lo stato emotivo del visitatore sezione per sezione:

ARRIVO (stato: [emozione iniziale — es: scettico, curioso, annoiato])
  ↓
[Sezione 1]: "Oh, [cosa pensa]" → stato cambia in [nuova emozione]
  ↓
[Sezione 2]: "[cosa pensa]" → stato cambia in [nuova emozione]
  ↓
...continua per ogni sezione...
  ↓
AZIONE: stato finale = [emozione che porta all'azione]

REGOLA: non cercare di saltare da "scettico" a "compra" in un step.
Ogni sezione sposta l'emozione di UN gradino. Il percorso è:
scettico → curioso → interessato → convinto → pronto → agisce
```

## FASE 8 — Anti-Pattern Detection

```
PATTERN AI DA EVITARE:
1. Hedging bilanciato ("on one hand... on the other hand")
   → FIX: prendi posizione
2. Transizioni troppo lisce
   → FIX: paragrafi corti, frasi singole, salti abrupti
3. Vocabolario uniforme
   → FIX: varia le parole per lo stesso concetto
4. Superlativi generici ("impressive", "significant")
   → FIX: il numero È il superlativo
5. Frase lista di cose bandite specifiche per il settore
   → Compilala dalla ricerca: quali frasi i competitor usano troppo?

PATTERN MANIPOLATIVI DA EVITARE:
1. Fake scarcity ("solo 3 posti rimasti")
   → Usa urgenza basata sulla realtà del mercato, non promozioni
2. Dark pattern nel pricing (nascondere il prezzo, auto-renewal aggressivo)
   → Trasparenza totale = vantaggio competitivo (i competitor lo fanno male)
3. Testimonial inventati
   → Usa numeri verificabili invece di testimonial
4. Fear-mongering
   → Loss framing basato su FATTI, non su paure inventate
```

## FASE 9 — Metriche di validazione

```
Per ogni gate della Conversion Architecture, definisci una metrica:

GATE 1 (Comprehension):
→ Metrica: bounce rate homepage < [X]%
→ Se più alto: il hero non comunica abbastanza velocemente

GATE 2 (Proof):
→ Metrica: scroll depth > [X]% raggiunge la sezione pricing
→ Se più basso: le sezioni proof non convincono

GATE 3 (Value):
→ Metrica: click rate sui CTA pricing > [X]%
→ Se più basso: il pricing framing non funziona

GATE 4 (Action):
→ Metrica: signup completion rate > [X]%
→ Se più basso: troppa frizione nel form

Monitora queste 4 metriche. Se una è sotto target,
il problema è in quel gate specifico — non nel design, non nel layout,
ma nel COPY che attiva il meccanismo psicologico di quel gate.
```

## FASE 10 — Output finale

Il framework produce questi deliverable:

```
1. DESIGN SPEC (docs/superpowers/specs/YYYY-MM-DD-[progetto]-copy-strategy-design.md)
   - Brand identity + positioning
   - Competitive landscape (tabella)
   - Target segments (3)
   - 3 Pilastri psicologici
   - Tone of voice + banned phrases
   - Trust architecture (5 layer)
   - Page-by-page strategy
   - Naming strategy
   - Academic/data references

2. PSYCHOLOGICAL FRAMEWORK ([progetto]-psychological-framework.md)
   - Core insight
   - 3 pilastri con meccanismo cognitivo dettagliato
   - Trust architecture spiegata
   - Conversion architecture (4 gate)
   - Trigger per pagina (tabella)
   - Emotional journey map
   - Anti-pattern list
   - Metriche di validazione

3. COMPLETE COPY ([progetto]-complete-copy.md)
   - Copy production-ready per ogni pagina
   - Ogni sezione con testo esatto
   - CTA text per ogni bottone
   - Microcopy (form labels, error messages, empty states)
   - Footer + disclaimer

4. COPY PER PAGINA ([progetto]-copy-pages-N-M.md)
   - File separati per gruppo di pagine
   - Più facili da consultare durante l'implementazione
```
